# Contexto

E-commerce D2C de materiais de construção, mercado brasileiro.
Público: [construtores/autônomos/consumidor final]
Stack: [Elixir/Phoenix + Ash framework]

# Requisitos de domínio (NÃO genéricos)

- Calculadora de consumo por categoria (tinta/m², piso/área+perda, argamassa)
- CEP + cálculo de frete antes do carrinho (peso volumétrico)
- Filtros por atributo TÉCNICO: bitola, espessura, classe, dimensão, marca
- Ficha técnica visível, comparação entre SKUs similares
- Busca tolerante a typo e a sinônimo técnico (ex: "vergalhão" = "barra de aço")

# Performance

- LCP < 2.5s em 4G, imagens otimizadas, lazy load
- Carrinho persistente, checkout em ≤ 3 telas

# Restrições

- LGPD para dados de cadastro
- aceita PIX, boletoe cartão via gateway de pagamento
- STOP gate: gere primeiro a árvore de componentes e o design system
  (tokens, tipografia, grid) antes de qualquer página. Aguarde aprovação.
